## RFC Operating Model

Every significant Harbor Vault feature should have a Request for Comments document before it is built.

RFCs live in `03 - RFCs/`.

RFCs should define:

- Problem statement
- Goals
- User stories
- UI direction or mockup notes
- Workflow
- Permissions
- Data model implications
- Harbor AI behavior
- Future considerations
- Acceptance criteria
